# Contributing

1. Read `AGENTS.md` and the owning source-of-truth document.
2. Keep each change inside one architectural boundary when practical.
3. Never commit credentials, cookies, browser profiles, or private provider exports.
4. Add contracts and tests before enabling external side effects.
5. Run `npm run check` before a commit.
6. Explain migrations, new secrets, and deployment changes in the pull request.

Commit messages should be short, imperative, and scoped, for example: `feat(runner): add fake Shopee discovery connector`.
